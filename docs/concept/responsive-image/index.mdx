# Responsive images

## Give the browser images to pick

Use the `srcset` attribute.

```jsx title= showLineNumbers
<img srcset="<path to image 1> <image 1 width>w, <path to image 2> <image 2 width>w, ..."  />

<img srcset="250-dog.png 250w, 500-dog.png 500w>
```

This syntax gives the browser two image choices to pick from.

First image, `250-dog.png`, has 250 pixels intrinsic width.

Second image, `500-dog.png`, has an intrinsic width of 500 pixels.

The browser calculates the appropriate space for an image and then picks the right one to display.

<Note>

Images' [intrinsic size](https://developer.mozilla.org/en-US/docs/Glossary/Intrinsic_Size) is the real size / dimensions of an image file in pixels

## Device pixel ratio

The number describes how many "real" pixels (the physical pixels on the screen) are used to display one "logical" pixel (a size set by software, such as CSS).

One logical pixel (`1px` in CSS) is not always equal to one physical pixel on the screen.

Older devices have fewer physical pixels, so software is written to fit the pixels those devices have.

Newer devices have more physical pixels, so they can display much sharper content.

But a newer device screen that has 1000 x 1000 physical pixels cannot render 500 x 500 logical-pixel content without leaving blank spaces.

So it treats 1 x 1 logical pixels as 2 x 2 physical pixels, which makes the content appear sharper while remaining compatible with the written layout.

That is an example of a 2x display. The device pixel ratio is 2 (`1:2` logical pixels to physical pixels).




## Appropriate space

When the browser reserves space for an image, it calculates that space by taking the image width (set by the developer, defaulting to `100vw`) and multiplying it by the device pixel ratio.

It then picks the first image that is equal to or larger than the reserved space.

For example:

```jsx title= showLineNumbers
<img srcset="250-dog.png 250w, 500-dog.png 500w, 1000-dog.png 1000w">
```

On 1X device (device pixel ratio = 1):

- when an image would take 1000 logical px in width, the reserved space is `1000 * 1 = 1000`, so `1000-dog.png` is displayed
- when an image would take 500 logical px in width, `500-dog.png` is displayed
- when an image would take 501 logical px in width, `1000-dog.png` is displayed

On 2X device:

- when an image would take 1000 logical px in width, the reserved space is `1000 * 2 = 2000`, so `1000-dog.png` is displayed
- when an image would take 250 logical px in width, `500-dog.png` is displayed

<Info>

The browser takes screen size, pixel density, zoom level, screen orientation, and network speed into account when reserving space.

</Info>




## Set amount of space that image would take

The `sizes` attribute tells the browser how much space the image will take.

```jsx title= showLineNumbers
<img
  srcset="250-dog.png 250w, 500-dog.png 500w, 1000-dog.png 1000w"
  sizes="
  (width <= 250px) 100px,
  (width <= 500px) 300px,
  70vw"
>
```

This means

- when the viewport is less than or equal to 250px, the image will take 100px
- when the viewport is less than or equal to 500px, the image will take 300px
- otherwise, the image will take 70vw

<Note>

The browser matches the first condition that is true, so order matters.

</Note>

## Picture element

```jsx title= showLineNumbers
<picture>
  <source
    type="image/avif"
    sizes="(width<= 1000px) 150px, 800px"
    srcset="300.avif 300w, 500.avif 500w, 700.avif 700w"
  />
  <source type="image/png" srcset="300.png 300w, 500.png 500w, 700.png 700w" />
  <img src="cat.png" />
</picture>
```
