# OpenAPI Document

An OpenAPI document is a text file, commonly called `openapi.json` or `openapi.yaml`, representing a JSON object, in either JSON or YAML format.

This file is called the <Color color="var(--secondary-font-color)"> root document </Color> and it can be split into multiple JSON or YAML files, for clarity.

## Minimal Document Structure

To be entirely precise, an OpenAPI document is a single JSON object containing fields following the structure defined in the [OpenAPI Specification (OAS)](https://spec.openapis.org/oas/v3.1.0).

The root object in any OpenAPI document is the [OpenAPI Object](https://spec.openapis.org/oas/v3.1.0#openapi-object) and <Color color="var(--secondary-font-color)"> only two of its fields are mandatory </Color>

- `openapi`
- `info`

Additionally, <Color color="var(--secondary-font-color)"> at least one </Color> of `paths`, `components` and `webhooks` <Color color="var(--secondary-font-color)"> is required </Color>.

- `openapi`(string): This indicates the version of the OAS this document is using, e.g. “3.1.0”. Using this field tools can check that the document correctly follow the spec.
- `info` ([Info Object](https://spec.openapis.org/oas/v3.1.0#info-object)): This provides general information about the API (like its description, author and contact information) but the only <Color color="var(--secondary-font-color)"> mandatory fields are </Color> `title` and `version`.
  - `title`(string): A human-readable <Color color="var(--secondary-font-color)"> name for the API </Color>, like “GitHub REST API”, <Color color="var(--not-require-but-good-to-know-font-color)"> useful to keep API collections organized </Color>.
  - `version`(string): Indicates the version of the API document (not to be confused with the OAS version above). <Color color="var(--not-nesary-for-now-future-font-color)"> Tools can use this field to generate code that ensures that clients and servers are interacting through the same version of the API </Color>.
- `paths`([Paths Object](https://spec.openapis.org/oas/v3.1.0#paths-object)): This describes <Color color="var(--secondary-font-color)"> all the endpoints </Color> of the API, including their parameters and all possible server responses. Server and client code can be generated from this description, along with its documentation.

### Example

```yaml title= showLineNumbers
openapi: 3.1.0
info:
  title: A minimal OpenAPI document
  version: 0.0.1
paths: {} # No endpoints defined
```
