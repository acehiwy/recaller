OpenAPI provides two mechanisms to specify input data, parameters and request body (message payload).

&#8203;<span style={{color: "var(--not-require-but-good-to-know-font-color)"}}> Parameters are typically used to identify a resource, whereas the message payload provides content for that resource. </span>

![Parameters And Operation Payload](assets/parameters-and-operation-payload.png)

<Info>

The edges marked with an asterisk(\*) are arrays.

</Info>

## Parameter Object

The `parameters` field in the [Path Item](https://spec.openapis.org/oas/v3.1.0#path-item-object) and [Operation Objects](https://spec.openapis.org/oas/v3.1.0#operation-object) is an array containing [Parameter Objects](https://spec.openapis.org/oas/v3.1.0#parameter-object).

When <span style={{color: "var(--secondary-font-color)"}}> provided in the Path Item Object </span>, the <span style={{color: "var(--secondary-font-color)"}}> parameters are shared by all operations on that path </span> (which can override individual parameters at the Operation Object level but not remove them).

Each Parameter Object describes one parameter with the following mandatory fields:

- `in` (string): [Location of the parameter](#parameter-location)
- `name` (string): Case-sensitive. <span style={{color: "var(--primary-font-color)"}}> Must be unique in each location </span>.

Additional optional fields include:

- `description` (string): Useful for documentation. Might contain usage examples, for instance.
- `required` (boolean): Whether this parameter must be present or not. The default value is false.

### Parameter Location

Parameters can reside in different locations, indicated by the `in` field. The most common ones are:

#### `path`

The parameter is <span style={{color: "var(--secondary-font-color)"}}> part of the route </span> of this operation.

The <span style={{color: "var(--secondary-font-color)"}}> parameter's name must appear in the path as a [template expression](https://spec.openapis.org/oas/v3.1.0#path-templating) </span>, i.e., delimited by curly braces `{}`.

For example, the path `/users/{id}` must contain at least one parameter described with:

``` yaml lines highlight={6-7}
paths:
  /users/{id}:
    get:
      parameters:
        - name: id
          in: path
          required: true
```

<Warning>

When using `path` parameters, the `required` field must be present and its value must be `true`.

</Warning>

#### `query`

The parameter is appended to the <span style={{color: "var(--secondary-font-color)"}}> query string </span> part of the operation's URL.

For example, the URL `/users?id=1234` can be parsed using:

``` yaml lines highlight={6}
paths:
  /users:
    get:
      parameters:
        - name: id
          in: query
```

#### `header`

The parameter is sent in a custom HTTP header as part of the request.

<Note>

Header names are case-<span style={{color: "var(--secondary-font-color)"}}>in</span>sensitive.

</Note>

### Parameter Type

Most of the time a parameter's type can be specified by using a [Schema Object](https://spec.openapis.org/oas/v3.1.0#schemaObject) in the schema field.

``` yaml lines highlight={4-7}
parameters:
  - name: id
    in: query
    schema:
      type: integer
      minimum: 1
      maximum: 100
```

<Info>

The [Content of Message Bodies](message-body-content#schema-object) page describes Schema objects in greater detail.

</Info>

In more advanced scenarios the `content` field can be used instead.

It provides a <span style={{color: "var(--secondary-font-color)"}}> single-entry map </span>(map that have only 1 key-value) of Media Types(as a key) to [Media Type Objects](https://spec.openapis.org/oas/v3.1.0#media-type-object)(as a value).

<Note>

&#8203;<span style={{color: "var(--secondary-font-color)"}}>Exactly one</span> of `schema` or `content` must be present. They cannot appear at the same time.

</Note>

## Request Body Object

&#8203;<span style={{color: "var(--not-require-but-good-to-know-font-color)"}}> When updating a record on a database, the parameters are typically used to identify the record whereas the message body provides its new content. </span>

The [Request Body Object](https://spec.openapis.org/oas/v3.1.0#request-body-object) is specified through the `requestBody` field in the [Operation Object](https://spec.openapis.org/oas/v3.1.0#operationRequestBody).


``` yaml lines highlight={4-5}
paths:
  /board:
    put:
      requestBody: 
        ...
```

<Info>

The only mandatory field in the Request Body Object is `content` which is described in detail in the [Content of Message Bodies](message-body-content#schema-object) page.

</Info>

<Note>

The Request Body Object also has a `description` string and a `required` boolean to state whether the message payload is mandatory.

</Note>

&#8203;<span style={{color: "var(--not-require-but-good-to-know-font-color)"}}> As a reminder, the snippet below describes an operation with a JSON request body containing a single integer with values between 1 and 100. </span>

``` yaml lines
requestBody:
  content:
    application/json:
      schema:
        type: integer
        minimum: 1
        maximum: 100
```

## <span style={{color: "var(--not-require-but-good-to-know-font-color)"}}> Example </span>

The [Tic Tac Toe sample API](https://oai.github.io/Documentation/examples/tictactoe.yaml) contains two endpoints

- one without parameters or request body (`/board`)
- another one with both (`/board/{row}/{column}`).


``` yaml lines
paths:
  # Single square operations
  /board/{row}/{column}:
    parameters:
      - name: row
        in: path
        required: true
        schema:
          type: integer
          minimum: 1
          maximum: 3
      - name: column
        in: path
        required: true
        schema:
          type: integer
          minimum: 1
          maximum: 3
    get:
      summary: Get a single board square
      responses: ...
    put:
      summary: Set a single board square
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: string
              enum: [".", "X", "O"]
      responses: 
        ...
```
