# Injection Context

is a place that u have access to `inject` function

## Valid injection context

### Class

With component clas, service class or directive class (class that decorated with `@Component`, `@Injectable` or `@Directive`)

Either inside `constructor` code block or class properties initialization statement

```typescript title= showLineNumbers
class Foo {
  private service1 = inject(service1);

  constructor() {
    const something = inject(something);
  }
}
```

### Factory function

function pass to `useFactory` of a [`Provider`](https://angular.dev/api/core/Provider) or <Color color="var(--can-not-understand-font-color)">an [`@Injectable`](https://angular.dev/api/core/Injectable)</Color>

or factory function for [`InjectionToken`](https://angular.dev/api/core/InjectionToken)

### API that run in an injection context

Some API are run inside injection context so it is valid to use `inject`

Eg. router guards `CanActivateFn`

```typescript title= showLineNumbers
const canActivateTeam: CanActivateFn = (route, state) => {
  // highlight-next-line
  return inject(someservice);
};
```

## Manually run code block in injection context

Wrap code block with [`runInInjectionContext`](https://angular.dev/api/core/runInInjectionContext)

```typescript title= showLineNumbers
@Injectable()
export class HeroService {
  // highlight-next-line
  private environmentInjector = inject(EnvironmentInjector);

  someMethod() {
    // highlight-start
    runInInjectionContext(this.environmentInjector, () => {
      inject(SomeService);
    });
    // highlight-end
  }
}
```

<Admonition type="note" title="note">

[`runInInjectionContext`](https://angular.dev/api/core/runInInjectionContext) require [`EnvironmentInjector`](https://angular.dev/api/core/EnvironmentInjector)

</Admonition>
