# Image Optimization

Built in image optimization feature 


&#8203;import [NgOptimizedImage](https://angular.dev/api/common/NgOptimizedImage) to the component
```typescript title= showLineNumbers
import { NgOptimizedImage } from '@angular/common'

imports: [
  NgOptimizedImage,
  // ...
],
```

use `ngSrc` instead of `src`
```jsx title= showLineNumbers
<img ngSrc="cat.jpg">
```

## Set image `fetchpriority` to `high` 

use `priority` attribute

```jsx title= showLineNumbers
<img ngSrc="cat.jpg" priority>
```




## Miscellaneous

`fetchpriority` attribute tell browser that the resource should be load before other resource with default or `low` `fetchpriority`

<Admonition type="tip" title="tip"> 

`fetchpriority=high` should not be overused, marking many resources with `high` make them not `high` at all

It should be use for LCP, typically 1-3 resources.

</Admonition>

`img` `loading=lazy` tell browser NOT to load image when it outside of the viewport

Browser load image when the image is about to be visible in viewport


```jsx title= showLineNumbers
<picture>
  <source type="image/avif"  sizes="(width<= 1000px) 150px, 800px" srcset="300.avif 300w, 500.avif 500w, 700.avif 700w"/>

  <source type="image/png" srcset="300.png 300w, 500.png 500w, 700.png 700w"/>
  <img src="cat.png" width="100%">
</picture>
```
`sizes="(<at this screen width>): <reserve this much space for an image>"`
so you can find the appropriate image file from `srcset` that fit my reserved space
