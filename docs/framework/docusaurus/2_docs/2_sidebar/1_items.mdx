# Sidebar Items

## Type of sidebar

- **[Doc](#doc)**: link to a doc page, associating it with the sidebar
- **[Link](#link)**: link to any internal or external page
- **[HTML](#html)**: renders pure HTML in the item's position
- **[Category](#category)**: creates a dropdown of sidebar items
- **[Autogenerated](./autogenerated)**: generate a sidebar slice automatically
- **[Ref](./multiple-sidebars#the-ref-item)**: link to a doc page, without making the item take part in navigation generation

## Doc: link to a doc {#doc}

Use the doc type to link to a doc page and assign that doc to a sidebar:

```typescript title= showLineNumbers
type SidebarItemDoc =
  // Normal syntax
  | {
      type: "doc";
      id: string;
      label: string; // Sidebar label text
      className?: string; // Class name for sidebar label
      customProps?: Record<string, unknown>; // Custom props
    }

  // Shorthand syntax
  | string; // docId shortcut
```

### Example

```javascript title=sidebars.js showLineNumbers
module.exports = {
  mySidebar: [
    // Normal syntax:
    {
      type: "doc",
      id: "doc1", // document ID
      label: "Getting started", // sidebar label
    },

    // Shorthand syntax:
    "doc2", // document ID
  ],
};
```

An item with type doc can be simply a string representing its ID:

<TabItem value="longHand" label="Longhand">

```javascript title=sidebars.js showLineNumbers

    {
      type: "doc",
      id: "myDoc",

```

</TabItem>
<TabItem value="shorthand" label="Shorthand">


</TabItem>

If you use the doc shorthand or [autogenerated sidebar](./autogenerated)
you would lose the ability to customize the sidebar `label` through item definition.

You can, however, use the `sidebar_label` Markdown front matter within that doc, which has higher precedence over the `label` key in the sidebar item.
Similarly, you can use `sidebar_custom_props` to declare custom metadata for a doc page.

## Link: link to any page {#link}

Use the `link` type to link to any page (internal or external) that is not a doc.

```typescript title= showLineNumbers
type SidebarItemLink = {
  type: "link";
  label: string;
  href: string;
  className?: string;
};
```

Example:

```javascript title=sidebars.js showLineNumbers
module.exports = {
  myLinksSidebar: [
    // External link
    {
      type: "link",
      label: "Facebook", // The link label
      href: "https://facebook.com", // The external URL
    },

    // Internal link
    {
      type: "link",
      label: "Home", // The link label
      href: "/", // The internal path
    },
  ],
};
```

## HTML: render custom markup {#html}

Use the html type to render <Color color='var(--secondary-font-color)'> custom HTML sidebar </Color> within the item's `<li>` tag.

:::info

html value will be rendered inside `li` tag

:::

This can be useful for inserting custom items such as dividers, section titles, ads, and images.

```typescript title= showLineNumbers
type SidebarItemHtml = {
  type: "html";
  value: string;
  defaultStyle?: boolean; // Use default menu item styles
  className?: string;
};
```

Example:

```javascript title=sidebars.js showLineNumbers
module.exports = {
  myHtmlSidebar: [
    {
      type: "html",
      value: '<img src="sponsor.png" alt="Sponsor" />', // The HTML to be rendered
      defaultStyle: true, // Use the default menu item styling
    },
  ],
};
```

## Category: create a hierarchy {#category}

Use the category type to create a hierarchy ( expandable group ) of sidebar items.

```typescript title= showLineNumbers
type SidebarItemCategory = {
  type: "category";
  label: string; // Sidebar label text.
  items: SidebarItem[]; // Array of sidebar items.
  className?: string;

  // Category options:
  collapsible: boolean; // Set the category to be collapsible
  collapsed: boolean; // Set the category to be initially collapsed or open by default
  link: SidebarItemCategoryLinkDoc | SidebarItemCategoryLinkGeneratedIndex;
};
```

### Example

```javascript title=sidebars.js showLineNumbers
module.exports = {
  docs: [
    {
      type: "category",
      label: "Guides",
      collapsible: true,
      collapsed: false,
      items: [
        "creating-pages",
        {
          type: "category",
          label: "Docs",
          items: ["introduction", "sidebar", "markdown-features", "versioning"],
        },
      ],
    },
  ],
};
```

A category item can be represented by an object whose <Color color='var(--secondary-font-color)'> key is its label </Color>,
and the <Color color='var(--secondary-font-color)'> value is an array of subitems </Color>.

### Category links

With category links, <Color color='var(--secondary-font-color)'> clicking on a category can navigate you to another page </Color>.

#### Generated index page

You can auto-generate an index page that displays all the direct children of this category.
The slug allows you to customize the generated page's route, which defaults to `/category/[categoryName]`.

<details>

<summary> Example </summary>

```javascript title=sidebars.js showLineNumbers
module.exports = {
  docs: [
    {
      type: "category",
      label: "Guides",
      // highlight-start
      link: {
        type: "generated-index",
        title: "Docusaurus Guides",
        description: "Learn about the most important Docusaurus concepts!",
        slug: "/category/docusaurus-guides",
        keywords: ["guides"],
        image: "/img/docusaurus.png",
      },
      // highlight-end
      items: ["pages", "docs", "blog", "search"],
    },
  ],
};
```

result :

![](@site/docs/assets/generated-index-page.png)

</details>

#### Doc link

A category can link to an existing document.

```javascript title=sidebars.js showLineNumbers
module.exports = {
  docs: [
    {
      type: "category",
      label: "Guides",
      // highlight-next-line
      link: { type: "doc", id: "introduction" },
      items: ["pages", "docs", "blog", "search"],
    },
  ],
};
```

#### Embedding generated index in doc page

You can embed the <Color color='var(--secondary-font-color)'> generated cards </Color> list in a normal doc page as well,
as long as the doc is <Color color='var(--primary-font-color)'> used as a category index page </Color>.

To do so, you need to use the DocCardList component, paired with the useCurrentSidebarCategory hook.

```jsx title=a-category-index-page.md showLineNumbers
import DocCardList from '@theme/DocCardList';
import {useCurrentSidebarCategory} from '@docusaurus/theme-common';

In this section, we will introduce the following concepts:

<DocCardList items={useCurrentSidebarCategory().items}/>
```

#### Collapsible categories

We support the option to expand/collapse categories. Categories are collapsible by default, but you can disable collapsing with `collapsible: false`.

```javascript title=sidebars.js showLineNumbers
module.exports = {
  docs: [
    {
      type: "category",
      label: "Guides",
      items: [
        "creating-pages",
        {
          type: "category",
          // highlight-next-line
          collapsible: false,
          label: "Docs",
          items: ["introduction", "sidebar", "markdown-features", "versioning"],
        },
      ],
    },
  ],
};
```

To make all categories non-collapsible by default, set the sidebarCollapsible option in plugin-content-docs to false:

```javascript title=docusaurus.config.js showLineNumbers
module.exports = {
  presets: [
    [
      "@docusaurus/preset-classic",
      {
        docs: {
          // highlight-next-line
          sidebarCollapsible: false,
        },
      },
    ],
  ],
};
```

:::note

The option in sidebars.js takes precedence over plugin configuration

:::

#### Expanded categories by default

Collapsible categories are collapsed by default. If you want them to be expanded on the first render, you can set collapsed to `false`:

```javascript title=sidebars.js showLineNumbers
module.exports = {
  docs: {
    Guides: [
      "creating-pages",
      {
        type: "category",
        label: "Docs",
        // highlight-next-line
        collapsed: false,
        items: ["markdown-features", "sidebar", "versioning"],
      },
    ],
  },
};
```

global config

```javascript title=docusaurus.config.js  showLineNumbers
module.exports = {
  presets: [
    [
      "@docusaurus/preset-classic",
      {
        docs: {
          // highlight-next-line
          sidebarCollapsed: false,
        },
      },
    ],
  ],
};
```
