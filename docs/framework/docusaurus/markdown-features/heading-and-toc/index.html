Heading and Table of contents

Markdown headings​

You can use regular Markdown headings.

## Level 2 title

### Level 3 title

#### Level 4 title

Each Markdown heading will appear as a table of contents ( TOC ) entry.

Table of contents

Customizing heading level TOC​

By default, heading level 2 (##) and 3 (###) will be automatically appear as TOC.

You can config this behaviour by using

  • Front Matter
  • docusaurus.config.js ( Global )

With Front matter​

  • Use toc_min_heading_level for minimun heading level shown in TOC.
  • Use toc_max_heading_level for maximum heading level shown in TOC
---
toc_min_heading_level: 3
toc_max_heading_level: 5
---

# Foo Bar heading
tip

See Front matter API spec for more details

With docusaurus.config.js​

module.exports = {
  themeConfig: {
    tableOfContents: {
      minHeadingLevel: 2,
      maxHeadingLevel: 5,
    },
  },
};
tip

See configuration reference for more details

Heading IDs​

Each heading has an ID that can be automatically generated or explicitly specified . Heading IDs allow you to link to a specific document heading in Markdown or JSX:

[link](#heading-id)

<Link to="#heading-id">link</Link>

Autogenerated​

By default, Docusaurus will generate heading IDs for you, based on the heading text.

For example, ### Hello World will have ID hello-world.

Explicitly specified​

A special Markdown syntax lets you set an explicit heading id:

### Hello World \{#my-explicit-id}
tip

Use the write-heading-ids CLI command to add explicit IDs to all your Markdown documents.

warning

Generated heading IDs will be guaranteed to be unique on each page.

But if you use custom IDs , make sure each one appears exactly once on each page
