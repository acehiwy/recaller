## Markdown headings

You can use regular Markdown headings.

```markdown title= showLineNumbers
## Level 2 title

### Level 3 title

#### Level 4 title
```

Each Markdown heading will appear as a table of contents ( TOC ) entry.

<Accordion title="Table of contents">

![](/docs/assets/table-of-content.png)

</Accordion>

### Customizing heading level TOC

By default, heading level 2 (`##`) and 3 (`###`) will be automatically appear as TOC.

You can config this behaviour by using

- Front Matter
- `docusaurus.config.js` ( Global )

---

#### With Front matter

- Use `toc_min_heading_level` for minimun heading level shown in TOC.
- Use `toc_max_heading_level` for maximum heading level shown in TOC

```markdown title=foo.md showLineNumbers
---
toc_min_heading_level: 3
toc_max_heading_level: 5
---

# Foo Bar heading
```

<Tip>

See [Front matter API spec](https://docusaurus.io/docs/api/plugins/@docusaurus/plugin-content-docs#markdown-front-matter) for more details

</Tip>

---

#### With `docusaurus.config.js`

```javascript title=docusaurus.config.js showLineNumbers
module.exports = {
  themeConfig: {
    tableOfContents: {
      minHeadingLevel: 2,
      maxHeadingLevel: 5,
    },
  },
};
```

<Tip>

See [configuration reference](https://docusaurus.io/docs/api/themes/configuration#table-of-contents) for more details

</Tip>

### Heading IDs

Each heading has an ID that can be <span style={{color: "var(--secondary-font-color)"}}> automatically generated </span> or <span style={{color: "var(--secondary-font-color)"}}> explicitly specified </span>.
Heading IDs allow you to link to a specific document heading in Markdown or JSX:

```jsx title= showLineNumbers
[link](#heading-id)

<Link to="#heading-id">link</Link>
```

---

#### Autogenerated

By default, Docusaurus will generate heading IDs for you, based on the heading text.

For example, `### Hello World` will have ID `hello-world`.

---

#### Explicitly specified

A special Markdown syntax lets you set an explicit heading id:

```markdown title= showLineNumbers
### Hello World {#my-explicit-id}
```

---

<Tip>

Use the [write-heading-ids](https://docusaurus.io/docs/cli#docusaurus-write-heading-ids-sitedir) CLI command to add explicit IDs to all your Markdown documents.

</Tip>

<Warning>

<span style={{color: "var(--primary-font-color)"}}> Generated </span> heading IDs will be <span style={{color: "var(--secondary-font-color)"}}>
  guaranteed to be unique
</span> on each page.

But if you use <span style={{color: "var(--secondary-font-color)"}}> custom IDs </span>, make sure <span style={{color: "var(--secondary-font-color)"}}> each one appears exactly once on each page </span>

</Warning>
