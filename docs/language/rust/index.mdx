# Rust

## Hello World

### Create Hello World file

```rust title=main.rs showLineNumbers
fn main() {
    println!("Hello, world!");
}
```

### Compile

Use `rustc` to compile a Rust file.

```bash title=
rustc main.rs
```

It will output binary file `main`.

### Run

Executing the output binary file:

```bash title=
./main
```

The main function is special: it is always the first code that runs in every executable Rust program

<Admonition type="info" title="info">

`println!` calls a Rust macro. If it had called a function instead, it would be entered as println (without the !).

For now, you just need to know that using a ! means that you’re calling a macro instead of a normal function and that macros don’t always follow the same rules as functions.

</Admonition>

<Admonition type="info" title="info">

Rust is an ahead-of-time compiled language, meaning you can compile a program and give the executable to someone else, and they can run it even without having Rust installed.

</Admonition>

## Printing

```rust title=
println!("foo bar");
```

### Content replacement

Content replacement uses `{}` as placeholders, and the arguments passed to `println!` provide the values for those placeholders.

```rust title=
println!("Hello {} from {}", "World" , "Rust");
// Hello World from Rust
```

#### Specifying replacement position

Integer and named arguments can be used as placeholders.

```rust title=
println!("{0}, this is {1}. {1}, this is {0}", "Alice", "Bob");
// Alice, this is Bob. Bob, this is Alice
```

```rust title=
println!("{foo}, this is {bar}. {bar}, this is {foo}", foo="Alice", bar="Bob");
// Alice, this is Bob. Bob, this is Alice
```

#### Variable replacement

Variable name can be used as a placeholder

```rust title= showLineNumbers
let foo = 50;
let bar = 100;
println!("{foo} and {bar}", bar="bar from print argument");
// 50 and bar from print argument
```

### <Color color="var(--not-require-but-good-to-know-font-color)"> Formatting </Color>

```rust title= showLineNumbers
// Different formatting can be invoked by specifying the format character
// after a `:`.
println!("Base 10:               {}",   69420); // 69420
println!("Base 2 (binary):       {:b}", 69420); // 10000111100101100
println!("Base 8 (octal):        {:o}", 69420); // 207454
println!("Base 16 (hexadecimal): {:x}", 69420); // 10f2c
println!("Base 16 (hexadecimal): {:X}", 69420); // 10F2C


// You can right-justify text with a specified width. This will
// output "    1". (Four white spaces and a "1", for a total width of 5.)
println!("{number:>5}", number=1);

// You can pad numbers with extra zeroes,
// and left-adjust by flipping the sign. This will output "10000".
println!("{number:0<5}", number=1);

let number: f64 = 1.0;
let width: usize = 5;
println!("{number:>width$}");
```

To obtain user input and then print the result as output, we need to bring the io input/output library into scope. The io library comes from the standard library, known as std:

```rust title=
use std::io;
```

<Admonition type="tip" title="later" icon="">

By default, Rust has a set of items defined in the standard library that it brings into the scope of every program. This set is called the prelude, and you can see everything in it in the [standard library documentation](https://doc.rust-lang.org/std/prelude/index.html).

</Admonition>

If a type you want to use isn’t in the prelude, you have to bring that type into scope explicitly with a use statement.

# Create Variable

assigning value

```rust title=
let guess = String::new();
```

The `::` syntax in the `::new` line indicates that new is an <Color color="var(--secondary-font-color)"> associated function of </Color> the `String` type.

An associated function is a function that’s implemented on a type, in this case `String`.

`String::new()` returns a new empty instance of a [String](https://doc.rust-lang.org/std/string/struct.String.html)

Calling function of a module

```rust title= showLineNumbers
    io::stdin()
        .read_line(&mut guess)
        .expect("Failed to read line");
```

call the stdin function from the io module

<Admonition type="tip" title="tip">

If we hadn’t imported the io library with use std::io; at the beginning of the program, we could still use the function by writing this function call as std::io::stdin. The stdin function returns an instance of std::io::Stdin, which is a type that represents a handle to the standard input for your terminal.

</Admonition>

<Admonition type="danger" title="clean up">

Next, the line .read_line(&mut guess) calls the read_line method on the standard input handle to get input from the user. We’re also passing &mut guess as the argument to read_line to tell it what string to store the user input in. The full job of read_line is to take whatever the user types into standard input and append that into a string (without overwriting its contents), so we therefore pass that string as an argument. The string argument needs to be mutable so the method can change the string’s content.

</Admonition>

The `&` indicates that this argument is a reference, which gives you a way to let multiple parts of your code access one piece of data without needing to copy that data into memory multiple times.

References are <Color color="var(--primary-font-color)"> immutable </Color> <Color color="var(--secondary-font-color)"> by default </Color>.
Hence, you need to write `&mut` guess rather than `&guess` to make it mutable.

## Handling Potential Failure with Result

```rust title= showLineNumbers
    io::stdin()
        .read_line(&mut guess)
        // highlight-next-line
        .expect("Failed to read line");
```

`read_line` return [Result](https://doc.rust-lang.org/std/result/enum.Result.html)

<Admonition type="tip" title="later" icon="">

If the `read_line` method returns an `Err`, it would likely be the result of an error coming from the underlying operating system.

</Admonition>

<Admonition type="tip" title="later" icon="">

If you don’t call `expect`, the program <Color color="var(--secondary-font-color)"> will compile </Color>, but you’ll get a warning:

Rust warns that you haven’t used the `Result` value returned from `read_line`, indicating that the program hasn’t handled a possible error.

</Admonition>
