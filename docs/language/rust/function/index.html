Functions

fn main() {
    print_labeled_measurement(5, 'h');
}

fn print_labeled_measurement(value: i32, unit_label: char) {
    println!("The measurement is: {value}{unit_label}");
}
note

In function signatures, each parameter type must be declared.

note

Within the same scope, functions can be called regardless of their defining order.

info

Rust code uses snake case as the conventional style for function and variable names

Statements and Expressions​

  • Statements are instructions that perform some action and do not return a value .
  • Expressions evaluate to a resultant value.

Statements​

Statements do not return values.

let y = 6; is a statement.

let y = 6;

let y = 6; does not return value so you can not assign this statement to a variable

let x = (let y = 6);
// Syntax Error: `let` expressions are not supported here
// expected expression, found statement (`let`)

Expressions​

Expressions evaluate to a value .

Expressions can be part of statements. In statement let y = 6;, 6 is an expression.

A new scope block created with curly brackets is an expression:

fn main() {
    let y = {
        let x = 3;
        x + 1
    };
    // {
    //   let x = 3;
    //   x + 1
    // }
    // is an expression

    println!("The value of y is: {y}");
}
note

x + 1 is a implicit return and doesn't have a semicolon at the end.

Expressions do not include ending semicolons.If you add a semicolon to the end of an expression, you turn it into a statement and it will then not return a value.

Functions with return values​

fn main() {
    let x = plus_one(5);

    println!("The value of x is: {x}");
}

fn plus_one(x: i32) -> i32 {
    x + 1
    // x + 1 is a return value
    // note that return expression does not have a semicolon
}

In Rust, the return value of the function is synonymous with the value of the final expression in the block of the body of a function .

You can return early from a function by using the return keyword and specifying a value

note

If function have a return values. The return type must be declared using ->
