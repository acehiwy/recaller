# To panic or Not to panic

## Use panic when

- You’re making the decision that a situation is unrecoverable <span style={{color: "var(--primary-font-color)"}}> on behalf of </span> the calling code.
- Your code could end up in the bad state
  - The bad state is something that is unexpected, <span style={{color: "var(--primary-font-color)"}}> as opposed to </span> something that will likely <span style={{color: "var(--secondary-font-color)"}}> happen occasionally </span>, like a user entering data in the wrong format.
- Your code after this point needs to rely on <span style={{color: "var(--secondary-font-color)"}}> not being </span> in this bad state.
- Continuing could be insecure or harmful
- Your code behavior is only guaranteed if input meet particular requirement but calling code does not meet the requirements(invalid input).
- It is <span style={{color: "var(--secondary-font-color)"}}> not make sense </span> for calling code to <span style={{color: "var(--primary-font-color)"}}> explicitly </span> handle the error

## Use Result when

- Failure is expected
- Your code might fail in a way that it could recover from

---

<Tip>

Returning `Result` is a good default choice

</Tip>

<Note>

If you can ensure by manually inspecting the code that you’ll <span style={{color: "var(--secondary-font-color)"}}> never </span> have an `Err` variant, it’s perfectly acceptable to call `unwrap`,
and even better to document the reason you think you’ll never have an `Err` variant in the expect text.

Here’s an example:

```rust title= showLineNumbers
 use std::net::IpAddr;

 let home: IpAddr = "127.0.0.1"
     .parse() // parse method return Result but we know that hardcoded "127.0.0.1" is always valid
     .expect("Hardcoded IP address should be valid");
```

</Note>

## Miscellaneous

When you do not want users of your struct to write to a struct field but still want to allow reading, you can implement a `getter` method. The reverse is also true for a `setter`.

```rust title= showLineNumbers
pub struct Guess {
    // highlight-next-line
    value: i32, // this field is private
}

impl Guess {
    // highlight-next-line
    pub fn value(&self) -> i32 { // user can read value using this getter method
        self.value
    }
}

```
