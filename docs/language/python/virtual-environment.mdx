A self-contained directory that contains a specific version of the Python interpreter and a set of additional packages.

It is used to separate Python's global modules from a project's modules.

Each project can have its own Python version and its own modules without interfering with others.


## Create Virtual environment

```bash title= showLineNumbers
python -m venv <dir path>

# eg
python -m venv /foo
```

<Note> 

`<dir path>` is a directory that will contain a version of the Python interpreter and Python modules.

</Note>

<Note> 

`<dir path>` will be created if it does not exist.

</Note>

## Activate and deactivate virtual environment

```bash title= showLineNumbers
source <dir path>/bin/activate

# eg
source /foo/bin/activate
```

### Deactivate

While the virtual environment is active, run this command:
```bash title= showLineNumbers
deactivate 
```

## Install packages into virtual environment

After activating the virtual environment

```bash title= showLineNumbers
python -m pip install requests==2.6.0 pandas
```

## Uninstall packages in virtual environment

```bash title= showLineNumbers
python -m pip uninstall requests==2.6.0 pandas
```

## Show installed package information

```bash title= showLineNumbers
python -m pip show request
```

## List all packages installed in virtual environment

```bash title= showLineNumbers
python -m pip list
```
