# Working with file

```python title= showLineNumbers
with open('path/to/file', 'w', encoding="utf-8") as f
  file_content = f.read()


# check whether file is closed
f.closed
# True
```

First argument is the file name.

Second <span style={{color: "var(--secondary-font-color)"}}> optional </span> argument is a string describing how the file will be used:

optional `encoding` keyword argument, default is `utf-8`, is the name of encoding use to encode and decode the file

- `r`: open for reading (default)
  - file's pointer start at the beginning
- `w`: open for writing
  - remove all file content or create a file if does not exist.
  - file's pointer start at the beginning
- `x`: open for exclusive creation, failing if the file already exists
- `a`: open for writing. appending to the end of file 
  - create file if does not exist
  - all writes append to the end of the file regardless of the current pointer position


- `b`: binary mode
- `t`: text mode (default)
- `+`: open for updating (reading and writing)

the default mode is `r` (synonym of `rt`)

`w+` and `w+b` open and truncate the file. 
`r+` and `r+b` open the file with no truncation.

<Note> 

Mode `b`, contents are return as [`bytes`](https://docs.python.org/3/library/stdtypes.html#bytes) object

Mode `t`, contents are return as `str`

</Note>

<Note> 

`with` keyword ensure that file is closed after the process is done even if an exception is raised.

</Note>


<Info> 

When <span style={{color: "var(--secondary-font-color)"}}> reading </span> in text mode, the default is to convert platform-specific line endings (`\n` on Unix, `\r\n` on Windows) to just `\n`. 

When <span style={{color: "var(--secondary-font-color)"}}> writing </span> in text mode, the default is to convert occurrences of `\n` <span style={{color: "var(--secondary-font-color)"}}> back </span> to platform-specific line endings. 
This behind-the-scenes modification to file data is fine for text files, but will corrupt binary data like that in JPEG or EXE files. 
Be very careful to use binary mode when reading and writing such files.

</Info>

<Info> 

In append binary mode, `a+b`, Pointer position will be moved ahead as the amount of bytes written regardless of its position.

For example, if the pointer position is at 2 in a 10-byte file and you write 3 bytes to the end, resulting in a 13-byte file, the pointer moves from 2 to 5, not from 10 to 13.

</Info>

## Read file

### `read` method

```python title= showLineNumbers
with open('filepath') as f:
  # do something
  file_content = f.read()
```

The `<file>.read(<size>)` method returns a string or bytes object.

An optional numeric `size` argument can be provided to specify how many characters (in text mode) or bytes (in binary mode) are read and returned. By default, all file content is returned.

```python title= showLineNumbers
with open...:
  first_10_char = file.read(10)
  _11_to_20th_char = file.read(10)
```

### `readline` method

```python title= showLineNumbers
with open('foo.py') as f:
  line_1 = f.readline()
  line_2 = f.readline()

  # or loop over the rest lines
  for line in f:
    pass
```

`readline()` reads a single line from the file

a newline character (`\n`) is left at the end of the string

<Tip> 

Returning each line as a list

```python title= showLineNumbers
my_list = f.readlines()

# or
my_list = list(f)
```

</Tip>


<Note> 

`read` and `readline` method return empty string if the end of the file is reached.

</Note>

<Note> 

The return type of a read operation is a string. Numbers need to be cast to `int` after reading.

</Note>

## Write file

```python title= showLineNumbers
f.write('This is a test\n') 
# 15 

# write list to file
f.writelines(['what' , 'ever'])
```

<Note> 

The `write` method returns the number of <span style={{color: "var(--secondary-font-color)"}}> characters </span> written.

</Note>

The `write` method takes a string or bytes object. Other types of objects need to be converted first.

### Write JSON

```python title= showLineNumbers
arr = ['item1' , 'item2' , 3 , 3333 , (1,2,3,4)]
dict = {'k1' : 1 , 'k2' : [1,2,3,4] , 'k3' : "foo"}

with open('filepath' , 'w+') as f:
    json.dump(arr, f)
    json.dump(dict, f)
```

## Move file's pointer position

```python title= showLineNumbers
f.seek(<offset> : int, <whence>: 0 | 1 | 2)
```

move file pointer for `<offset>` <span style={{color: "var(--primary-font-color)"}}> byte </span> from `<whence>`

`whence` argument accept
- 0 (default): beginning of the file 
- 1: at current file pointer position
- 2: end of the file

Eg:
```python title= showLineNumbers
# move ahead 3 byte from the beginning
f.seek(3,0) 

# move ahead 3 byte from the current pointer position
f.seek(3,1)

# move back 3 byte from the end
f.seek(-3,2)

# move to the end of the file
f.seek(0,2)
```

<Note> 

The `seek` function should be used when you open a file in <span style={{color: "var(--secondary-font-color)"}}> binary mode </span>.

</Note>

<Danger> 

When you open a file <span style={{color: "var(--secondary-font-color)"}}> in text mode </span>, `open('file', 'r+t')`

`seek`'s argument should follow these otherwise error might be raised:
- the `offset` argument should be aligned with the value returned from `f.tell()`
- `whence` argument can only be `0`
- the only two exceptions are:
  - move to the beginning, `seek(0)`, of the file
  - move to the end, `seek(0,2)`, of the file



<Info> 

Reason: In text mode, the read operation should return valid character data. If you move the file pointer into the middle of the bytes that make up a character, there is no reliable way to know which character it is supposed to represent.
(If this part is unclear, read more about [character encoding](https://www.youtube.com/watch?v=GMF2Z1EZHXk).)

If you want to move a file pointer in text mode, you can use `read` instead. For example, to move four characters forward, use `read(4)`.

</Info>

</Danger>


## Miscellaneous

combine file paths:
```python title= showLineNumbers
from pathlib import Path

file_paths = ['/example' , 'file' , 'path' , 'file.txt' ]

path_obj = Path(*file_paths)
print(path_obj.as_posix())
# /example/file/path/file.txt


#or
path_obj = Path('base-path')
path_obj = path_obj.joinpath(*['file' , 'path']) # joinpath is pure function
print(path_obj.as_posix())
# base-path/file/path
```
