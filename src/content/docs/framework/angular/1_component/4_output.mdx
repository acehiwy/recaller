# Custom event

Component can define custom event it emit by using [`output`](https://angular.dev/api/core/output)

```typescript title=foo.component.ts showLineNumbers
@Component({
  /*...*/
})
export class Foo {
  bar = output<void>();
}
```

```jsx title=parent.component.ts showLineNumbers
<app-foo (bar)="dosomethingWithEvent($event)" />
```

<Aside type="note" title="info">

Angular custom events do not bubble up the DOM

</Aside>

## Emiting event

by calling `<output>.emit`

```typescript showLineNumbers
this.bar.emit("some value");
this.bar.emit({
  k1: 1,
  k2: 2,
});
```

## Consuming event

```typescript showLineNumbers
@Component({
  template: `<app-foo (bar)="dosomething($event)"></app-foo>`,
})
export class App {
  dosomething(valueFromBarEvent: string) {}
}
```

## Output alias

```typescript showLineNumbers
@Component({/*...*/})
export class CustomSlider {
  changed = output({alias: 'valueChanged'});
}
```

<Aside type="tip" title="tip"> 

Avoid prefixing output names with "on"

</Aside>

## <Color color="var(--not-require-but-good-to-know-font-color)">Subscribing to outputs programmatically</Color>

When creating component dynamically, you can subscribe to output by
```typescript showLineNumbers
const someComponentRef: ComponentRef<SomeComponent> = viewContainerRef.createComponent(/*...*/);

someComponentRef.instance.someEventProperty.subscribe(eventData => {
  console.log(eventData);
});
```

Angular automatically unsubscribe when component is destroyed

You can also manually unsubscribe
