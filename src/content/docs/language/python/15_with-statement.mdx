The `with` statement allows you to automatically execute the setup and teardown code blocks of a callable (class or function).

For a callable to have setup and teardown code blocks, it needs to support the [context manager](https://docs.python.org/3/reference/datamodel.html#context-managers) protocol.

In other words, it needs to have an `__enter__` method (for setup code) and an `__exit__` method (for teardown code).

```python showLineNumbers
with foo_callable() as return_value_from__enter__method:
  print('foo')

print('bar')
```
Executing the `with` statement on a context manager callable allows Python to:

1. execute the callable object `foo_callable`
1. execute `__enter__` method
  1. if the `with` statement has a target, such as `as return_value_from__enter__method`, it is assigned the value returned from `__enter__`.
1. execute the code inside the `with` block
1. execute the `__exit__` method at the end of the `with` block.
  1. the `__exit__` method is executed whether or not an exception occurs inside the `with` block
  1. if an exception occurs and is not handled by `__exit__`, the exception is propagated to the outer scope.


<Aside type="tip" title="tip"> 

You can pass multiple callables to `with`:
```python showLineNumbers
with (
    A() as a,
    B() as b,
):
  pass
```

equivalent to:
```python showLineNumbers
with A() as a:
    with B() as b:
        pass
```

</Aside>


<details>

<summary> `with` statement is sugar syntax for `try...finally` </summary>

Opening file with `with`:
```python showLineNumbers
with open("hello.txt", "w") as file:
  file.write("Hello, World!")
```

with `try...finally`:
```python showLineNumbers
file = open("hello.txt", "w")

try:
    file.write("Hello, World!")
finally:
    file.close()
```

</details>

## Class-based context manager

You can create a class that supports the context manager protocol.

```python showLineNumbers
class Foo:
  def __init__(self, arg1):
    self._arg1 = arg1
    print("Init with", arg1)

  def __enter__(self):
    print("Entering the context")
    return "Hello, World!"

  def __exit__(self, exception_type, exception_value, exception_traceback):
    print("Leaving the context")
    print(f"{exception_type  = }")
    print(f"{exception_value = }")
    print(f"{exception_traceback = }")


# highlight-start
with Foo('bar') as hello:
     print(hello)
# highlight-end

# Init with bar
# Entering the context
# Hello, World!
# Leaving the context
# exception_type  = None
# exception_value = None
# exception_traceback = None
```

<Aside type="note" title="note"> 

An exception raised inside a `with` block <Color color="var(--secondary-font-color)"> ends that block </Color>. [`with` ends when an exception occurs]

The exception is then passed to the `__exit__` method.

</Aside>

<Aside type="note" title="note"> 

If the `__exit__` method returns `True`, the exception is suppressed. Otherwise, the exception is propagated to the outer scope.

<Aside type="tip" title="tip"> 

This is useful when you want a `callable` to automatically handle a certain exception.

```python showLineNumbers
def __exit__(self, exception_type, exception_value, exception_traceback):
  if isinstance(exception_value, IndexError):
    # handle error
    return True
```

</Aside>

</Aside>

## Function-based context manager

```python showLineNumbers
from contextlib import contextmanager

# highlight-next-line
@contextmanager
def foo(arg1):
  print('executing inside generator arg1 =', arg1)
  try:
    # highlight-next-line
    yield "Hello, World!"
    print('after `with` enclosing end if exception does not occur')
  except BaseException as e:
    print('generator handle exception')
  finally:
    print("generator finally")


with foo('bar') as hello:
  print('executing inside with statement hello =', hello)
  raise Exception('something')
  print('not executed after exception raised')

print('end of program')

# output:
# executing inside generator arg1 = bar
# executing inside with statement hello = Hello, World!
# generator handle exception
# generator finally
# end of program
```

<Aside type="note" title="note"> 

A decorated function must be a generator function. It must `yield` <Color color="var(--primary-font-color)"> exactly one value</Color>.

</Aside>

<Aside type="note" title="note"> 

At the point where the generator yields, the code inside `with` statement is executed.

</Aside>

<Aside type="note" title="note"> 

An exception raised inside a `with` block <Color color="var(--secondary-font-color)"> ends that block </Color>. [`with` ends when an exception occurs]

The exception is then re-raised inside the generator at the point where `yield` occurs.

If the generator does not re-raise the exception passed to it, the exception is suppressed.

</Aside>

<Aside type="note" title="note"> 

If `with` ends without an exception, the code after `yield` resumes, [including the `finally` statement, of course]

</Aside>


## Async context manager

An async context manager is a context manager whose setup and teardown functions are async functions.

It use with `async with` statement.

```python showLineNumbers
async with foo() as bar:
  pass
```

### Class

use `__aenter__` for setup method

use `__aexit__` for teardown method

```python showLineNumbers
class AsyncContextManager:
    async def __aenter__(self):
        await log('entering context')

    async def __aexit__(self, exc_type, exc, tb):
        await log('exiting context')
```

<Aside type="tip" title="tip"> 

You can define both sync and async setup / teardown methods to support both `with` and `async with`

</Aside>

### Function

use `asynccontextmanager` decorator

```python showLineNumbers
from contextlib import asynccontextmanager

@asynccontextmanager
async def get_connection():
    conn = await acquire_db_connection()
    try:
        yield conn
    finally:
        await release_db_connection(conn)

async def get_all_users():
    async with get_connection() as conn:
        return conn.query('SELECT ...')
```
